CSS규칙

1. 파일경로

■ 절대경로사용

  • img 경로[src="/img/이미지.gif"]
  • css 경로[src="/com/css/스타일.css"]
  • js 경로[src="/com/js/스크립트.js"]

2. 문서내 css 사용

■ import css

css 상세구조는 폴더구조 참고

@import url ("cm.import.css"); - 기본 import css(cm.reset.css / cm.layout.css / cm.k7.css 기본포함. @import url ("cm.reset.css"); - 문서 초기화 스타일 @import url ("cm.layout.css"); - 공통 레이아웃 (ex. 타입별 레이아웃구조[1단 레이아웃, 2단 레이아웃 등], 뎁스별 타이틀 체계, gnb 등) @import url ("cm.k7.css"); - 공통 스타일 (ex. 버튼, 테이블, 게시판, 기본 블릿리스트 등)

■ CSS 기본사용 규칙

  • css 는 해당 카테고리의 것만 사용하며, 공통 css파일을 해당 카테고리 css 페이지 내에 import 시켜서 사용한다.
  • 페이지내 style 사용은 금하며, 예외적으로 input 또는 팝업(레이어팝업 포함) 사이즈 지정이 필요한 경우 style="width" 사용할 수 있다.
  • 문서내 margin/padding 사용이 필요할 경우 해당 요소의 클래스내에서 처리하며, 아래 예시와 같이 inner style 또는 공통클래스(ex. marT15, padL20 등)를 만들어 사용하지 않는다
    ex) style="margin-top:20px; padding-bottom:15px;" 또는 class="marT20 padL20" 등은 사용금지
  • 공통으로 사용되는 css 파일내의 스타일 선언은 상속을 최소화 하여 다른 채널에서 사용하게 되는경우나 위치변동 사항이 생겨도 쉽게 이식할수 있도록 함.
  • HTML 페이지내 최상위 div class="최상위 클래스로 약속된 값" 으로 함
  • 모든 속성은 영문소문자로만 작성하며, 숫자가 맨 처음 올수 없다.
  • 속성값으로 한글폰트 및 이미지 URL 표기시 작은 따옴표{''}를 사용하지 않는다.
    ex. font-family:맑은고딕, 돋움; background:url(/front/img/test.gif)
  • CSS 코드 작성시 들여쓰기를 하지 않는다.
  • 빈줄사용 : 의미있는 객체를 구분하기 위하여 코드 그룹간 1줄씩 빈줄을 넣을수 있다.
    단. 빈줄의 간격은 1줄을 초과하지 않게 한다.
  • 속성 마지막에는 반드시 (;) 선언.

3. 네임규칙

■ id/class 네이밍룰

  • id 네이밍룰 : 카멜표기법 사용 (단어와 단어의 사이를 대문자로 구분)
    ex. #subContainer #tabFaqList
  • class 네이밍룰 : 언더스코어(_) 표기법 (단어와 단어의 사이를 _로 구분)
    ex. .btn_reg .btn_edit
  • class 중 정렬 또는 칼라를 표현하는 네이밍의 경우 예외적으로 카멜표기법 사용
    ex. alL, alC, pointBl

4. 어트리뷰트 및 속성 선언순서

■ 〈link〉

  • rel, type, href 순서로 애트리뷰트를 선언한다
    ex. 〈link rel="stylesheet" type="text/css" href="css/import.css"〉

■ CSS 파일 내 속성선언 순서

display:; visibility:; float:; position:; width:; height:; margin:; padding:; overflow:; font-family:; font-size:; font-weight:; font-style:; line-height:; text-indent:; text-align:; border:; background:#color url() 0 0 repeat;
예약어 사용
순서 의미 대표되는 속성(그룹) 비고
1 표시 display, visibility
2 흐름 float 가상선택자 :after를 이용하여 해제 또는
가상선택자 사용이 불가피 할 경우 reset.css에 정의되어 있는 clfix를 이용하여 float 해제 사용
3 위치 position z-index 사용
4 크기 width / height
5 간격 margin / padding
6 넘침 overflow 플롯해제를 위해서 overflow 사용금지. 예외적 사항을 제외하고 js 핸들링을 위한 경우에만 사용권장.
7 폰트(그룹) font 폰트는 축약속성을 사용하지 않으며, 다음의 순서로 정의
font-family, font-size, font-weight, font-style, font-variant
8 테두리(그룹) border 아래 순서로 축약속성 사용
border-width, border-style, border-color
9 배경 background(그룹) 아래 순서로 축약속성 사용
background-color, background-image, background-position, background-repeat, background-attachment
10 기타속성 위에 언급되지 않은 나머지 속성들은 테두리 관련속성 이전에 선언하며, 기타 속성내의 선언순서는 무관함.
